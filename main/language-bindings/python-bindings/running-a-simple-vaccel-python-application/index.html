Running a simple vAccel Python application🔗

For a simple example of using vaccel-python, you can replicate in Python the image classification example from Running the examples.

Create a new Python file called classify.py with the following content:

classify.py
#!/usr/bin/python3

import sys

from vaccel.session import Session
from vaccel.image import ImageClassify


def main():
    sess = Session(flags=0)
    print(f'Session id is {sess.id()}')
    res = ImageClassify.classify_from_filename(sess, sys.argv[1])
    print(res)

if __name__=="__main__":
    main()

As with the original classify example, to execute classify.py you need to configure vAccel:

export VACCEL_PLUGINS=libvaccel-noop.so

Assuming vAccel in installed at /usr/local, running the Python classification should give you:

$ python3 classify.py /usr/local/share/vaccel/images/example.jpg
Loading libvaccel
Session id is 1
('This is a dummy classification tag!', 'This is a dummy imgname!')

export VACCEL_LOG_LEVEL=4

Looking at the log messages, you can see that in this case vAccel is being reloaded. This happens so the vAccel library's symbols can be properly exported when loaded from Python, to be used by the plugins. You can skip the redundant bootstrapping by setting VACCEL_BOOTSTRAP_ENABLED:

export VACCEL_BOOTSTRAP_ENABLED=0
