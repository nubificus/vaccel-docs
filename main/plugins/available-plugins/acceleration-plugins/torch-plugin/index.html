    Torch Plugin🔗

    The Torch plugin for vAccel implements basic PyTorch C++ API (LibTorch) support for vAccel operations.

    Supported operations🔗

    • Image classification
    • JIT loading and forwarding
    • Matrix-to-matrix multiplication

    Installing PyTorch C/C++ API files (LibTorch)🔗

    You can find instructions on how to install the required files here. The rest of this guide assumes PyTorch is installed at /opt/pytorch and its' libraries are at /opt/pytorch/lib.

    Installing the plugin🔗

    You can get the latest Torch plugin binary release from the Releases page of the vAccel repository. Releases include binaries for x86_64/aarch64 Ubuntu-based systems.

    Requirements🔗

    The prebuilt Torch plugin binaries depend on libstb. You can install it with:

    sudo apt install libstb0

    TAR🔗

    To install the TAR binary package of the latest Torch plugin release:

    wget https://github.com/nubificus/vaccel/releases/download/v0.7.1/vaccel-torch_0.2.0_amd64.tar.gz
# Replace '/usr/local' below with the desired installation prefix
tar xfv vaccel-torch_0.2.0_amd64.tar.gz --strip-components=2 -C /usr/local
# Update pkg-config files with the correct prefix
find /usr/local -name "vaccel-torch.pc" -exec sed -i 's:^\(prefix=\).*:\1/usr/local:g' {} \;

    wget https://github.com/nubificus/vaccel/releases/download/v0.7.1/vaccel-torch_0.2.0_arm64.tar.gz
# Replace '/usr/local' below with the desired installation prefix
tar xfv vaccel-torch_0.2.0_arm64.tar.gz --strip-components=2 -C /usr/local
# Update pkg-config files with the correct prefix
find /usr/local -name "vaccel-torch.pc" -exec sed -i 's:^\(prefix=\).*:\1/usr/local:g' {} \;

    Latest artifacts🔗

    You can also find prebuilt artifacts of the latest Torch plugin revision at:

    # TAR
https://s3.nbfc.io/nbfc-assets/github/vaccel/plugins/torch/rev/main/x86_64/release/vaccel-torch-latest-bin.tar.gz

    # TAR
https://s3.nbfc.io/nbfc-assets/github/vaccel/plugins/torch/rev/main/aarch64/release/vaccel-torch-latest-bin.tar.gz

    Usage🔗

    To specify Torch plugin as the selected plugin for vAccel execution:

    export VACCEL_PLUGINS=libvaccel-torch.so

    Ensure LibTorch, vAccel and the Torch plugin libraries are in the library search paths before trying to use the plugin.

    To run image classification you will also need to specify a file with labels using VACCEL_TORCH_LABELS. vAccel includes an file imagenet labels file. Assuming vAccel is installed at /usr/local, you can specify the file with:

    export VACCEL_TORCH_LABELS=/usr/local/share/vaccel/labels/imagenet.txt

    If vAccel is installed using the DEB package (at /usr) you can omit the variable.

    Running an example🔗

    Export the necessary variables:

    export VACCEL_PLUGINS=libvaccel-torch.so
export VACCEL_TORCH_LABELS=/usr/local/share/vaccel/labels/imagenet.txt
# Optionally, for verbose output
export VACCEL_LOG_LEVEL=4

    Assuming vAccel is installed at /usr/local, you can run an image classification with a ResNet model from https://s3.nbfc.io/torch/resnet18.pt with:

    $ classify /usr/local/share/vaccel/images/example.jpg 1 \
      https://s3.nbfc.io/torch/resnet18.pt
