TVM Plugin

The TVM plugin for vAccel implements image inference vAccel operations with TVM.

Supported operations

  • Image classification

Install TVM C/C++ API files

You can find instructions on how to install the required files here. The rest of this guide assumes TVM is installed at /opt/tvm and its' libraries are at /opt/tvm/build.

Install vAccel

Information about vAccel installation can be found here. The rest of this guide assumes vAccel is installed at /usr/local.

Install the plugin

Download the plugin for x86_64:

https://s3.nbfc.io/nbfc-assets/github/vaccel/plugins/tvm/rev/main/x86_64/release/vaccel-tvm-latest-bin.tar.gz

or aarch64:

https://s3.nbfc.io/nbfc-assets/github/vaccel/plugins/tvm/rev/main/aarch64/release/vaccel-tvm-latest-bin.tar.gz

and extract to the desired installation directory, ie. to install in /usr/local:

sudo tar xfv vaccel-tvm-latest-bin.tar.gz --strip-components=2 -C /usr/local

Run an example

After having installed TVM, vAccel and the TVM plugin you are ready to run an image classification example.

Ensure TVM libraries are in the library search path:

export LD_LIBRARY_PATH=/opt/tvm/build

If vAccel and/or the TVM plugin are installed in a non-standard library path you will also need to include them in the LD_LIBRARY_PATH.

Define the TVM plugin as the selected plugin for vAccel execution:

export VACCEL_PLUGINS=libvaccel-tvm.so

Alternatively, you can provide the full path to the plugin library, without needing to have the plugin in your library search path:

export VACCEL_PLUGINS=/usr/local/lib/x86_64-linux-gnu/libvaccel-tvm.so

Configure the vaccel log level for more verbose output:

export VACCEL_LOG_LEVEL=4

And finally run an image classification with a ResNet model from https://s3.nbfc.io/models/tvm/x86_64/resnet18-v2-7.so:

$ classify /usr/local/share/vaccel/images/example.jpg 1 \
